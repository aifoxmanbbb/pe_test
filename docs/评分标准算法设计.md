# 评分标准算法设计

## 1. 目标

设计可配置、可版本化、可审计的评分引擎，支持：

- 仅录入成绩时自动换算评分成绩。
- 初中体考、高中体考、体测等不同业务规则。
- 各省份、各年份红头文件导入后的统一执行。
- 导入识别结果人工确认后生效。

## 2. 输入输出定义

## 2.1 输入

- `biz_type`：`pe_exam` / `fitness_test`
- `standard_id`：评分标准版本 ID（批次绑定）
- `item_id`：项目 ID
- `gender`：性别
- `grade_stage`：学段/年级阶段
- `raw_value`：成绩数值
- `raw_text`：成绩文本（如 `2:03.4`）

## 2.2 输出

- `score_value`：评分成绩（门槛项可为空）
- `pass_flag`：是否及格
- `excellent_flag`：是否优秀
- `full_flag`：是否满分
- `hit_rule_id`：命中规则 ID
- `trace`：计算追踪 JSON（归一化值、命中规则、匹配策略）

## 3. 成绩标准化

所有输入先转换为可比较的 `normalized_value`：

- 时间：`mm:ss.S` -> 秒（如 `2:03.4 -> 123.4`）
- 距离：`2.39m` -> `2.39`
- 次数：`185次` -> `185`
- 枚举门槛：`合格/不合格` -> `1/0`

解析失败返回 `RAW_VALUE_PARSE_ERROR`。

## 4. 规则模型

## 4.1 规则类型

- `discrete_map`：精确映射
- `segment_step`：分值段阶梯（成绩区间 -> 分值）
- `interval_linear`：区间线性
- `formula`：表达式计算
- `gate_only`：门槛判定

## 4.2 方向定义

- `higher_better`：成绩越大越好（如跳远、跳绳、实心球）
- `lower_better`：成绩越小越好（如跑步计时）

## 4.3 无精确命中策略

默认策略：`floor_by_direction`（就低不就高）

- `higher_better`：取不大于输入值的最大规则点
- `lower_better`：取不小于输入值的最小规则点

无可用规则点时返回最低规则分（或配置默认分）。

## 5. 计算流程

1. 按 `biz_type + standard_id + item_id + gender + grade_stage` 加载规则。
2. 成绩标准化。
3. 按规则类型执行计算。
4. 计算阈值状态（及格/优秀/满分）。
5. 写入成绩、命中规则、追踪 JSON。

## 6. 业务规则补充

## 6.1 体考门槛项

- 门槛项可独立判定并记录。
- 门槛项不及格时，后续项目仍允许录入。
- 不做“阻断标记”或“拦截标记”。

## 6.2 体测项目

- 支持项目按性别与年级阶段差异计分。
- 支持分值段规则（成绩区间对应分值）。

## 7. 红头文件导入设计

1. 上传 PDF/Excel。
2. 创建识别任务：`PENDING -> PARSING -> REVIEW -> PUBLISHED`。
3. 结构化输出：标准头、项目表、规则表。
4. 人工确认后发布。

模板三段结构：

- `standard_meta`：地区、年份、业务类型、学段、版本号
- `standard_item`：项目、性别、阈值、计分模式
- `standard_rule`：分值段/映射/公式

## 8. 伪代码

```text
function calcScore(input):
  rules = loadRules(
    biz_type=input.biz_type,
    standard_id=input.standard_id,
    item_id=input.item_id,
    gender=input.gender,
    grade_stage=input.grade_stage
  )

  value = normalize(input.raw_text, input.raw_value)

  if rules.type == gate_only:
    passed = evalGate(rules, value)
    return result(score=null, pass_flag=passed, trace={value, rule: rules.id})

  score, hitRule = matchRule(rules, value, strategy='floor_by_direction')
  return result(
    score=score,
    pass_flag=score >= rules.pass_score,
    excellent_flag=score >= rules.excellent_score,
    full_flag=score >= rules.full_score,
    hit_rule_id=hitRule.id,
    trace={value, rule: hitRule.id, strategy:'floor_by_direction'}
  )
```

## 9. 异常与审计

- 规则缺失：`RULE_NOT_FOUND`
- 规则冲突：`RULE_CONFLICT`
- 成绩超界：`RAW_VALUE_OUT_OF_RANGE`
- 成绩解析失败：`RAW_VALUE_PARSE_ERROR`

审计要求：

- 所有换算记录 `rule_trace_json`。
- 不提供“按批次重算历史成绩”；历史变化通过“复制历史标准生成新版标准”演进。

## 10. 可优化项（待确认）

1. 识别规则增加置信度阈值自动拦截。
2. 单位词典扩展（米/m、秒/s、次/count）适配多省文件。
3. 标准详情增加规则冲突可视化比对。
