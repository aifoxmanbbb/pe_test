# 体考学生成绩管理系统二开总设计文档

## 0. 已确认决策（2026-04-02）

- 评分采用双轨：成绩 + 评分成绩并存。
- 创建测试批次时必须手工选择一个评分标准版本。
- 每次创建测试批次都需要重新选择标准（不能沿用批次历史自动复算）。
- 评分标准来源支持：红头文件识别导入（PDF）、手工录入、表格导入。
- 标准导入识别结果必须人工确认后才能发布。
- 历史标准可复制生成新版标准；标准版本号必填。
- 暂不保存“微调版本镜像”，微调即修改当前标准。
- 规则冲突默认策略固定为“就低不就高”。
- 多教练模式：班级可绑定多个教练。
- 家长端并入“学生角色（学生本人视角）”，仅允许查看权限内数据。
- 报表导出仅做同步导出，不做异步任务队列。
- 新增体质测试后采用：**同一规则引擎 + 双业务板块（体考/体测）**。

---

## 1. 总体目标

基于当前仓库 `kinit-admin + kinit-api + kinit-task` 输出可直接开发的方案，覆盖：

- 菜单与页面信息架构
- 前端 UI 基线（含横竖线 ASCII 线框）
- 后端模块结构与接口分层
- 标准化评分算法与自动算分
- SQL 初始化脚本与数据模型
- 角色权限与数据隔离

---

## 2. 统一业务口径（本期强约束）

## 2.1 术语与展示

- 全局统一术语：`及格 / 优秀 / 满分`。
- 颜色统一：
  - 及格/及格率：黄色
  - 不及格：红色
  - 优秀/优秀率：绿色
  - 满分/满分率：纯黑色
- 表格与图形展示成绩时，必须同时展示阈值对比：`及格阈值 / 优秀阈值 / 满分阈值`。

## 2.2 搜索与筛选

- 全部页面移除“时间范围”筛选。
- 搜索统一支持“学生关键信息”：姓名/学号/联系方式关键字。
- 主筛选顺序统一：`批次 -> 学校 -> 年级 -> 班级`。

## 2.3 性别与学段

- 阈值与评分规则必须支持性别维度（男/女）。
- 同一项目在不同地区/年份/性别下允许不同规则。
- 初中与高中通过页面 Tab 隔离展示和筛选。

---

## 3. 体考与体测的关系

## 3.1 结论

- **录入分开设计**：体考录入、体测录入两个入口。
- **图表分开设计**：体考看板和体测看板分开，不混用口径。
- **计算引擎共用**：都走“成绩 -> 分值/等级”的规则匹配。

## 3.2 统一引擎字段（核心）

- `biz_type`：`pe_exam`（体考） / `fitness_test`（体测）
- `item_id`
- `gender`
- `grade_stage`
- `rule_version_id`
- `raw_value`
- `score_value`
- `level`（及格/优秀/满分）

## 5. 前端落地（kinit-admin）

- 体考目录：`src/views/Vadmin/PE/`（沿用当前）
- 体测目录：`src/views/Vadmin/Fitness/`（新增）
- 路由按 `biz_type` 完整隔离。
- 公共组件与图表组件复用，不复用业务口径。

---

## 6. 后端落地（kinit-api）

新增业务模块建议：`apps/vadmin/sport/`

- `models/`：通用标准、规则、批次、成绩
- `service/rule_engine.py`：统一评分引擎
- `service/standard_import.py`：规则导入解析
- `views_exam.py`：体考接口
- `views_fitness.py`：体测接口

关键策略：

- 引擎按 `biz_type` 分发规则集。
- 体考和体测分别维护标准版本。
- 一套导入框架，两套模板。

---

## 7. 数据模型与 SQL

在现有 `kinit-api/scripts/sql/20260401_pe_system_init.sql` 基础上，新增/调整：

- 标准主表增加 `biz_type`。
- 规则表增加 `biz_type + grade_stage` 维度。
- 批次表增加 `biz_type`。
- 成绩表增加 `biz_type`。

若不改旧表，也可新增体测平行表，但推荐统一主表以减少重复开发。

---

## 8. 权限模型与数据权限

角色与范围：

- 管理员：全局数据权限
- 学校主管（校长/领导）：仅本校数据权限
- 学校老师/教练：仅本班数据权限
- 学生：仅本人查看权限

权限码按业务拆分：

- 体考：`pe.*`
- 体测：`fitness.*`

---

## 9. 已确认补充规则

1. 门槛项不及格时，后续项目允许录入，且不额外标记。
2. 良好率不纳入体考统计与展示。
3. 标准发布后允许直接修改。
4. 规则冲突默认策略固定为“就低不就高”。
