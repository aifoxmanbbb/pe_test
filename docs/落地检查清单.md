## 待补强（下一轮建议）

1. **标准导入链路**：PDF/Excel/手工录入的解析与人工确认流程尚未落数据库流程，"导入 PDF""导入表格""手工录入"三个按钮为占位状态。

2. **批量导入成绩**：录入页"批量导入"按钮为占位，需接入 Excel 模板上传 → 解析 → 人工确认 → `score/upsert` 流程。

3. **报表导出**：当前"生成报表"按钮为前端模拟，未生成真实 Excel/PDF 文件。需对接 `/vadmin/fitness/reports/*/export` 和 `/vadmin/pe/reports/*/export` 接口并实现文件流下载。

4. **学生角色前端入口**：后端已有 `analysis/student/self` 接口；前端尚未增加学生视角独立路由与菜单绑定。

5. **批次管理页面**：目前批次通过 API 创建，缺少独立的批次管理 UI（创建/编辑/状态流转：draft → ongoing → finished）。建议在体考/体测管理下各补一个"批次管理"子菜单。

6. **动态学生选项**：录入页的学生下拉目前为静态样例，需接入真实 `/vadmin/auth/dept/students` 或学生名册接口，支持姓名/学号模糊搜索。

7. **TypeScript 全量检查**：现有系统历史 TS 问题需独立一轮工程化清理。
