# 年级对比分析 UI设计文档

## 1. 页面定位

- 菜单：体考管理 / 年级对比分析
- 目标：单个年级下各班平均分、各项目平均成绩/平均分与历史趋势对比
- 页面类型：筛选 + KPI + 多图表 + 班级列表

## 2. 路由与权限

- 路由：`/pe/analysis/grade`
- 组件：`views/Vadmin/PE/Analysis/Grade/Grade`
- 权限码：`pe.analysis.grade`

## 3. 页面结构

- 顶部 Tab：`初中 | 高中`
- 筛选区：批次、学校、年级
- 主体 Card（查询按钮下方）：
  - 主标题：`某批次-某学校-年级 体育项目情况分析`
  - KPI：年级平均分、及格率、优秀率、满分率
  - 图表：
    - 各班平均分对比
    - 各班项目均值对比（平均成绩+平均分）
    - 各班历史批次平均分趋势
  - 列表：班级成绩列表

## 4. 展示规则

- 页面仅支持单个年级展示。
- 列表中各项目（含门槛项）单元格上下展示`平均成绩/平均分`。
- 图表必须体现阈值参考线或阈值图例。
- 颜色统一：及格黄、不及格红、优秀绿、满分黑。

## 5. 交互规则

- 移除筛选项：项目、性别、时间范围。
- 图表标题短化，主 Card 标题居中加粗且字号更大。
- 班级列表去外框。

## 6. 接口清单

- `GET /vadmin/pe/analysis/grade/kpi`
- `GET /vadmin/pe/analysis/grade/class-avg`
- `GET /vadmin/pe/analysis/grade/class-item-avg`
- `GET /vadmin/pe/analysis/grade/class-history-trend`
- `GET /vadmin/pe/analysis/grade/class-list`

## 8. 状态处理

- 年级未维护班级：提示先维护班级基础数据。
- 样本不足：展示“样本偏少，结论仅供参考”。
